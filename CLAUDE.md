## 1) General Working Principles

### 1.1) FILE LIMIT

You are allowed to edit a maximum of 10 files at a time, regardless of the operation. For implementations with 10 or more files, you must divide the plan into batches of 10 files each.

### 1.2) NO INVENTING

If there is a lack of information or references in any operation, inventing is prohibited. You must stop the operation and ask the user.

### 1.3) PLAN FIRST, THEN CODE

Before generating code, you must do the following;

- Prepare a file list (which files will be changed/added/deleted + why)

- If applicable, a matrix of new dependencies (Which library, version + why)

- Present the plan; never start implementation without approval.
